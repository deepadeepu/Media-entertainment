# StreamVault README

## Overview
**StreamVault** is a unified content discovery and personalization platform developed by the **DeepaG Media Group Product Division**. It serves as a Product Requirements Document (PRD) showcase, outlining the strategy for integrating disparate media streaming services into a single, cohesive user experience.

The project is designed to solve "subscription fatigue" by providing a centralized hub for content discovery, watchlist management, and personalized recommendations across various streaming silos.

## Key Features
* **Unified Content Discovery**: A single interface to search and browse content from multiple streaming providers.
* **Personalization Engine**: Advanced metrics and user personas (e.g., "The Infinite Scroller," "The Content Curator") drive the recommendation logic.
* **Interactive Roadmap**: A phased development plan ranging from MVP (Core Search) to Version 2.0 (Social Watch Parties).
* **Comprehensive Project Frameworks**: Includes detailed User Stories, Acceptance Criteria, RACI matrices, and Risk Management logs.

## Project Structure
The project is contained within a single, highly stylized HTML document (`index.html`) that uses modern CSS and JavaScript for a premium, "internal-use-only" corporate aesthetic:
* **Floating Navigation**: Allows quick access to different PRD sections like Executive Summary, Personas, and Metrics.
* **Modern UI Components**: Utilizes CSS variables, grid/flexbox layouts, and Intersection Observer API for smooth scroll animations.
* **Print-Optimized**: Includes specific `@media print` styles for professional document generation.

## Technical Stack
* **HTML5/CSS3**: Custom-designed with a "Navy & Gold" premium theme.
* **Typography**: Leverages Google Fonts (*Cormorant Garamond*, *DM Sans*, and *DM Mono*).
* **JavaScript**: Pure JS used for UI/UX enhancements and scroll-linked navigation indicators.

## Usage
To view the StreamVault PRD:
1. Open `index.html` in any modern web browser.
2. Use the floating navigation on the right to jump between PRD sections.
3. For a physical copy, use the browser's print function to generate a formatted PDF.
